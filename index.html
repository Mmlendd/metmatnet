# Обновление репозиториев и установка BIND
apt-get update && apt-get install -y bind9 bind9-utils

После установки редактируем файлы зон. Пример прямой зоны ssa.lan и обратной 168.192.in-addr.arpa:

🔹 2. Файл resolv.conf и системный резолвер

На клиентских машинах (HQ-CLI, BR-SRV) необходимо указать корректный DNS-сервер. По заданию — 192.168.10.2.

nano /etc/resolv.conf
# Содержимое:
nameserver 192.168.10.2
# (при необходимости можно добавить 8.8.8.8 для fallback)
⚠️ В некоторых дистрибутивах /etc/resolv.conf перезаписывается. Рекомендуется также проверить настройки в /etc/systemd/resolved.conf или использовать chattr +i как временное решение.

Если nslookup info.ssa.lan не работает — принудительно пропишите 192.168.10.2 в resolv.conf на HQ-CLI.

🔹 3. Статические IP-адреса (интерфейсы)

Меняем настройки с DHCP на статику. Работаем с файлом /etc/netplan/*.yaml или (в старых системах) /etc/network/interfaces.
По вашим заметкам: правим options, прописываем static и назначаем адреса.

# Пример для интерфейса ens19 (HQ-SRV / BR-SRV)
# /etc/netplan/01-netcfg.yaml
network:
version: 2
renderer: networkd
ethernets:
ens19:
dhcp4: no
addresses: [192.168.10.2/24] # для HQ-SRV
# для BR-SRV: 192.168.20.2/24
routes:
- to: default
via: 192.168.10.1 # шлюз (или 192.168.20.1)
📌 На HQ-CLI необходимо также «обезвредить» старый ipv4address (удалить конфликтующие настройки). Используйте rm /etc/network/ipv4address (если присутствует legacy-конфиг).
🔹 4. Маршрутизация и default gateway

На каждом сервере должен быть прописан маршрут по умолчанию.
Для HQ-SRV (192.168.10.2) default gateway 192.168.10.1.
Для BR-SRV (192.168.20.2) default gateway 192.168.20.1.

# Добавить временный маршрут:
ip route add default via 192.168.10.1 dev ens19
# Проверить таблицу маршрутизации
ip route show

Если после перезагрузки маршрут пропадает — зафиксировать в /etc/netplan (раздел routes) или в файле /etc/network/interfaces: gateway 192.168.10.1.

🧪 Проверка: ping 8.8.8.8 или ping 192.168.10.1 для теста связанности со шлюзом.
🔹 5. Конфигурация BR-SRV, HQ-CLI и роутеры

🌐 BR-SRV и HQ-SRV

На BR-SRV аналогично настраиваем ens19: IP 192.168.20.2/24, шлюз 192.168.20.1.
Также важно синхронизировать /etc/resolv.conf (nameserver 192.168.10.2). Возможно потребуется на роутере пробросить DNS или настроить форвардинг.

# На BR-SRV (или HQ-CLI) проверить наличие статики в опциях
# Пример для Debian-стиля: /etc/network/interfaces
auto ens19
iface ens19 inet static
address 192.168.20.2
netmask 255.255.255.0
gateway 192.168.20.1
dns-nameservers 192.168.10.2
💡 «Первый модуль обезврежен» — после применения корректных статических адресов и удаления конфликтующих конфигураций сеть стабильна.

🔄 Настройка роутеров (Hq-router)

По скриншотам из методички: на роутере HQ требуется задать статические маршруты между сетями 192.168.10.0/24 и 192.168.20.0/24, а также проверить трансляцию DNS-запросов.

# Пример команды на роутере (Cisco-like / Linux netfilter)
ip route add 192.168.20.0/24 via 192.168.10.2
iptables -A FORWARD -i ens19 -o ens20 -j ACCEPT (пример)
🔹 6. Диагностика: nslookup и возможные проблемы

🔍 2. Проверка связи с DNS

Убедиться, что сервер доступен
🔍 3. Статус BIND на сервере
systemctl status bind9
Проверить active (running)
🔍 4. Тест dig

Прямая проверка резолвера

✨ Доп. совет: на роутере иногда требуется разрешить DNS-прокси или добавить перенаправление. Если проблема остаётся — возможно в файле options.conf (Bind) разрешить рекурсивные запросы для подсетей 192.168.10.0/24.
